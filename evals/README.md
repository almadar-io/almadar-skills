# Almadar Skills Evals

Simple evaluation framework for testing the quality of Orbital schemas generated by the kflow-orbitals skill.

## Quick Start

```bash
# Run all evals
pnpm eval

# Run specific category
pnpm eval:basic
pnpm eval:edge

# Run with specific model
pnpm eval --model gpt-4o

# Output as JSON
pnpm eval --format json > report.json

# Compare two reports
pnpm eval:compare report-a.json report-b.json
```

## Architecture

The eval pipeline:

1. **Generate** - LLM generates orbital schema from natural language prompt
2. **CLI Validate** - `npx @almadar/cli validate` checks structural correctness
3. **Semantic Check** - LLM-as-judge evaluates whether schema meets expectations
4. **Report** - Aggregate results and generate pass/fail report

## Directory Structure

```
evals/
├── types.ts           # Type definitions
├── semantic.ts        # LLM-as-judge implementation
├── runner.ts          # Orchestration logic
├── cli.ts             # CLI interface
├── cases/
│   ├── basic/         # Simple CRUD apps
│   ├── multi-entity/  # Multi-entity with relations
│   ├── workflows/     # Multi-step flows
│   └── edge-cases/    # Tricky prompts
└── baselines/         # Saved reports for comparison
```

## Adding Test Cases

Create a JSON file in `cases/`:

```json
{
  "id": "my-test-case",
  "description": "Brief description",
  "prompt": "Build an app that...",
  "semanticExpectations": [
    "Has Entity X with fields Y and Z",
    "Uses pattern P for feature F"
  ],
  "tags": ["basic", "crud"]
}
```

That's it! The runner will auto-discover it.

## Scoring

- **CLI validation fails** → score = 0, test fails
- **CLI passes** → score = semantic_score (0-1 from LLM judge)
- **Pass threshold** → score >= 0.6 (3/5 or better)

## Options

```bash
-f, --filter <pattern>     Glob pattern for case files
-t, --tags <tags>          Comma-separated tags to filter
-m, --model <model>        Model to use
-c, --concurrency <N>      Max concurrent cases (default: 3)
--format <text|json|md>    Output format
```

## Examples

```bash
# Run only basic tests
pnpm eval --tags basic

# Run with GPT-4
pnpm eval --model gpt-4o

# Generate JSON report
pnpm eval --format json > baselines/$(date +%Y-%m-%d).json

# Compare today vs yesterday
pnpm eval:compare baselines/2025-02-08.json baselines/2025-02-09.json
```

## CI Integration

```yaml
# .github/workflows/eval.yml
- name: Run evals
  run: pnpm --filter @almadar/skills eval --format json > report.json
  env:
    OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}

- name: Upload report
  uses: actions/upload-artifact@v4
  with:
    name: eval-report
    path: report.json
```
